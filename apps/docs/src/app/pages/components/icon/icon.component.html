Icon

The Icon component (gp-icon) is the only supported way to render icons across the system.

Overview

Icons help clarify actions and improve scanability. Always render them with gp-icon to keep the system consistent and accessible.

Usage

Use icons to reinforce meaning and improve scannability. Icons should support text and actions, not replace them, especially when the meaning is not universally understood.

Do not rely on icons alone for critical actions or complex concepts. When clarity matters, always pair icons with text labels or accessible descriptions.

Icons may be used on their own only when their meaning is widely recognized (such as search, close, or delete) and appropriate accessibility labels are provided.

Use icons consistently across the interface to avoid ambiguity, and prefer a limited, system-defined set of icon names rather than introducing new ones ad hoc.

Show codeHide code 
<gp-icon name="search" ariaLabel="Search" />
<gp-icon name="filter" ariaLabel="Filter" />
<gp-icon name="add" ariaLabel="Add" />

Icon names

Icon names are system-defined. Use only the supported names below (for example: add, search, filter, delete, arrow-right, bell).

API

Property Values Required
name System icon name Yes
size sm | md | lg No
decorative boolean No
ariaLabel string Required when decorative = false

Accessibility

Decorative icons must use decorative="true". If an icon conveys meaning, provide ariaLabel. Do not rely on color alone to convey meaning.

Implementation note

Icons are currently implemented using Iconoir. This is an internal implementation detail and may change in the future. The public API remains stable.
