Icon

The Icon component (gp-icon) is the system-supported way to render icons in Percepta-owned UI components. Framework components provided by PrimeNG may use their own icon system.

Overview

Icons help clarify actions and improve scanability. Always render them with gp-icon to keep the system consistent and accessible.

Icon ownership

Percepta uses two icon systems with different responsibilities.

System-owned icons
Use gp-icon for icons that belong to Percepta-owned UI components, such as buttons, icon-only actions, and custom interface elements. These icons are part of the design system API and must use system-defined names.

Framework-owned icons
Components provided by PrimeNG (such as inputs, selects, date pickers, and overlays) use PrimeNGâ€™s own icon system. These icons are managed by the framework to ensure correct layout, alignment, and accessibility, and must not be replaced by gp-icon.

Do not mix icon systems within the same component.

Usage

Use gp-icon only in system-owned components. Do not use gp-icon inside PrimeNG components.

Use icons to reinforce meaning and improve scanning. They should support text and actions, not replace them.

Do not rely on icons alone for critical actions or complex concepts. When clarity matters, pair them with text or an accessible description.

Use standalone icons only when the meaning is universal (search, close, delete) and accessible labels are provided.

Be consistent and limit names to the system-defined set to avoid ambiguity.

<gp-icon name="search" ariaLabel="Search" />
<gp-icon name="filter" ariaLabel="Filter" />
<gp-icon name="add" ariaLabel="Add" />

Icon names

Icon names are system-defined. Use only the supported names below (for example: add, search, filter, delete, arrow-right, bell).

API

Property Values Required
name System icon name Yes
size sm | md | lg No
decorative boolean No
ariaLabel string Required when decorative = false

Accessibility

Decorative icons must use decorative="true". If an icon conveys meaning, provide ariaLabel. Do not rely on color alone to communicate.

Implementation note

Icons are implemented with Iconoir. This is an internal detail and may change in the future; the public API remains stable. This implementation applies only to gp-icon and does not affect framework-owned icons.
