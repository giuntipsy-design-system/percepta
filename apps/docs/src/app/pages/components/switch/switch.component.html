Switch

Switches represent a binary decision that turns a setting on or off.

Use a checkbox when the user is selecting items in a list or making multiple independent selections.

Preview

Switches toggle immediately between on and off. Labels should always clarify the outcome.

Usage

Switches represent immediate, binary changes. They are designed for situations where a setting takes effect as soon as the user toggles it, without requiring an additional confirmation step.

A switch communicates an active or inactive state at a glance and is best suited for persistent preferences, modes, or features that can be safely enabled or disabled instantly.

When to use switches

  • For settings that apply immediately. The effect of the change should be visible as soon as the switch is toggled.
  • For enabling or disabling a single feature, mode, or behavior.
  • For independent binary decisions that do not affect other options.
  • When the control reflects a persistent state rather than a one-time action.
  • When a concise label can clearly describe the affected property.

When to use something else

  • Use a checkbox when a change requires an explicit confirmation or submission.
  • Use checkboxes for multiple independent selections within a group.
  • Use radio buttons for mutually exclusive choices.
  • Use a button when an action is destructive or should not be applied immediately.
  • Avoid switches when an intermediate or mixed state is required.

Labeling guidelines

  • Describe the feature or behavior being controlled, not the current state.
  • Keep labels short and unambiguous, ideally one or two words.
  • Avoid labels that require the user to infer what “on” or “off” means.

Keyboard support

  • Tab moves focus to the switch.
  • Space toggles the switch on or off.
  • The state changes immediately on toggle.

Accessibility

Provide a visible, descriptive label for every switch. Native binary semantics and ARIA are handled by the framework.
