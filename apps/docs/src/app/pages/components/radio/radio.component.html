Radio Button

Single selection control for choosing one option within a group.

Radio buttons are used when users must select a single option from a short, visible set. Use radios for immediate comparison, checkboxes for independent choices, and selects for long lists that benefit from collapsing.

Usage

Radio buttons are designed for explicit, visible choices. They are best suited for situations where all available options should remain visible and the user must commit to exactly one selection.

Radios support quick comparison and make the selected state explicit at all times, helping users understand the decision they are making before interacting.

When to use radio buttons

  • Options are mutually exclusive. Only one value can be selected within the group.
  • Comparison matters. Seeing all options at once helps users evaluate differences.
  • The set of options is limited. Radio buttons work best with short, manageable lists.
  • A sensible default exists. Preselect an option when it reflects a safe or known choice.
  • Structure improves clarity. Vertical layouts work best for groups with more than two options.

When to consider alternatives

  • Use a checkbox for binary decisions. On/off or yes/no choices are communicated more clearly with checkboxes.
  • Use a segmented control for short sets. When space is limited and options are few, segmented controls provide a compact alternative.
  • Use a select for longer lists. As option sets grow, collapsed or searchable controls scale better.
  • Avoid radios for large sets. Long lists reduce scanability and slow decision-making.

Labeling guidelines

  • Keep labels concise and specific. Radio labels should be short and easy to scan.
  • Avoid ambiguity. Each option should be self-explanatory without relying on external context.

Grouping

Radio buttons must be presented as a group. The group is the semantic unit, and each option shares the same name so the selection is mutually exclusive.

Keyboard support

  • Tab focuses the selected radio within the group.
  • Arrow keys move focus and selection within the group.
  • Space selects the focused option.
  • Focus remains inside the group while navigating options.

Accessibility

Radio groups follow native semantics. Ensure every option has a visible label and each group has a clear legend or context so assistive technologies can announce the choices properly.
