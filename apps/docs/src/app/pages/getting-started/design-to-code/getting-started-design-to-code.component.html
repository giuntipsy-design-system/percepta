How Percepta works

Percepta connects design decisions to production UI through tokens, shared components, and documented usage patterns.

Design reference

Visual foundations and brand guidelines live at design.giuntipsy.com. This repo only documents technical implementation.

Technical overview

Tokens are the single source of truth for colors, typography, spacing, and states. They are compiled and consumed by shared Angular components.

PrimeNG provides the base primitives. Percepta reuses their structure and overrides styles, tokens, and behaviors where needed.

Governance & contribution

The system evolves through controlled changes to tokens and shared components. Changes can be proposed by the frontend team and reviewed before merging.

A breaking change alters component API, token meaning, or behavior in a way that requires updates in consuming apps. Use the project issue tracker to report bugs or propose improvements.
