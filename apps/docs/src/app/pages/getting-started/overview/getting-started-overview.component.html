Getting started

Percepta is the design system implementation layer for Giunti Psychometrics. This repository documents how design decisions are translated into code, including tokens, shared UI components, and canonical documentation.

It is intended to be the single reference point for frontend implementation, ensuring consistency, scalability, and long-term maintainability across products.

Overview

Percepta bridges the gap between design intent and frontend execution. It defines a structured system of design tokens, reusable Angular components, and documented interaction patterns that can be reliably consumed by applications.

Rather than prescribing visual styles ad hoc, Percepta establishes a contract between design and code: design decisions are encoded once and reused everywhere.

What Percepta solves

Percepta addresses common problems in large UI codebases:

  • Reduces drift between design and implementation over time
  • Standardizes UI behavior, spacing, typography, and interaction states
  • Avoids duplication and custom styling at application level
  • Provides a clear, shared source of truth for frontend teams

By centralizing these decisions, Percepta enables teams to move faster without sacrificing consistency.

Who this is for

This documentation is primarily intended for:

  • Frontend engineers working on Giunti Psychometrics products
  • Teams building interfaces with Angular and PrimeNG
  • Developers consuming shared components and tokens

Designers may also find this useful as a reference for how design decisions are realized in code.

Platforms and technologies

Percepta is currently built on:

  • Angular
  • PrimeNG
  • Sass-based design tokens

The system is designed to be technology-aware but not technology-bound, allowing future extensions if needed.
